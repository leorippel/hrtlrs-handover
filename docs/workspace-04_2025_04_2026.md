## `Kiwi_superT_project`
- **System:** *Actinidia chinensis* (kiwifruit), genotypes D/H/R, developmental stages 3–5, tissue IP/OP
- **What it is:** SuperTranscriptome assembly pipeline — STAR alignment → merged BAM →
  Trinity (both genome-guided and de novo assembly) → transcript renaming/merging →
  EvidentialGene (`tr2aacds.pl`) to collapse redundant transcripts → downstream DEG and
  DTU (differential transcript usage, via `swish`/fishpond) analysis across 45 contrasts
  (tissue, developmental stage, and between-genotype comparisons).
- **Key decision:** ROAST (a supertranscript-collapsing step normally used alongside Lace/
  SuperTranscripts) was **not** used — Leo chose the **No-ROAST path**. Downstream analysis
  folders are named accordingly (`analysis/DEG/noroast`, `analysis/DTU/noroast`). Anyone
  continuing this should know that decision was deliberate, not an oversight — worth
  confirming the reasoning with Leo before reversing it.
- **Status:** Active
- **Notes:** A `DTU_not_DEG.R` script cross-references DTU-significant genes against DEG
  results per contrast — useful for finding genes with isoform-level changes not visible
  in gene-level DEG, but the contrast-name mapping between DTU and DEG output files is done
  via regex pattern-matching (`dtu_to_deg()`), which is brittle — check it still matches if
  contrast naming conventions change.

## `Malus_Speed`
- **System:** *Malus domestica* (apple), multiple cultivars, Early/Late developmental stages,
  multiple tissues, staged by BBCH/Temporal/BS systems
- **What it is:** RNA-seq DEG analysis (Salmon → DESeq2) plus co-expression clustering
  (`coseq`) of DEG unions.
- **What's been done:** the original DESeq2 design pooled Early-vs-Late across cultivars,
  which raised a pseudoreplication concern (Early vs Late is confounded with cultivar
  identity when only 2–3 cultivars represent each stage group). The clustering step
  (`COseq_dblStrat.R`) addresses this by **stratifying independently by `Comp_type` × `Tissue`**
  (e.g. BBCH-staged samples and Temporal-staged samples are clustered separately, per tissue)
  rather than pooling incompatible staging systems together — each stratum gets its own DEG
  union, its own `coseq` run (kmeans, logclr transform, DESeq normalization), and its own
  cluster/profile/direction outputs.
- **Status:** Active
- **Notes:** `DEGbk` folder is a backup of an earlier DEG run — check before assuming `DEG`
  is the only/current version.

## `MM2M`
- **Collaborator(s):** Nigel and team (per QC.sh handover notes) — samples metadata for
  two experiments (high-low dry matter, high-resolution) supplied by them, with sample
  IDs kept matching their internal naming for cross-referencing.
- **System:** ⚠️ NEEDS CONFIRMATION (subfolders: `Apple_Biops_1-2`, `DelPerFruit`, `Empire`,
  `SuperStorage`, `workspace`) — likely apple ("Empire" is an apple cultivar), storage/
  postharvest-related given "DelPerFruit" and "SuperStorage" naming
- **What it is:** RNA-seq QC/processing pipeline (FastQC → Ribodetector rRNA removal →
  fastp → FastQC again) across three sequencing batches (128-sample, 192-sample,
  192-sample-batch-2).
- **What's been done / known issue — `SuperStorage`:** a library-size QC check on this
  batch turned up a clear problem cluster. Using ~9M read pairs/GB for PE150 vendor-cleaned
  gzipped FASTQ:
  - **Tier 1 — failed/unusable, drop or re-sequence:** samples 255, 254, 356, 389, 380, 332,
    374, 279 (1–107 MB, ~10K–1.0M reads) — far below any DEG floor; these would wreck
    DESeq2 size-factor normalization and inflate dispersion estimates.
  - **Tier 2 — marginal/risky (~1.4–2.5M reads):** 276, 259, 148, 362, 278, 223, 277
    (150–290 MB) — usable only with strong replication; expect noisy logFCs, inflated FDR.
    Flag, don't auto-drop — check PCA and saturation curves first.
  - **Tier 3 — low but workable (~3–5M reads):** 21, 15, P205, 413, 124, 453, 402
    (333–477 MB) — below ENCODE's 20–30M recommendation but adequate for moderate/high-
    expression genes; consider including depth as a covariate (or `voomWithQualityWeights`).
  - **Tier 4 — borderline acceptable (~5–7M reads):** 125, 127, 135, 136, 137, 253, 372,
    416, 171 (540–745 MB) — fine for standard bulk DEG, marginal for splicing/low-abundance work.
  - **Also flagged:** samples 383, 428, 119 had no R2 file in the <1GB listing at time of
    check — worth confirming these aren't truncated/still in-flight rather than genuinely broken.
  - **Caveat carried forward:** these sizes are **pre-ribodetector** (vendor-"clean") files.
    rRNA carryover in plant tissue (especially stressed tissue) varies a lot — Tier 2 samples
    in particular could drop into Tier 1 territory once rRNA is actually removed. Re-check
    post-ribodetector read counts before finalizing which samples to exclude.
- **Status:** Active
